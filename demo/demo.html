jquery.popups.js

A css driven jquery plug-in for all your popup and dialog needs

Compatible down to IE7

About

jquery.popups.js is a css driven popup and dialog creator. Though it can serve many other purposed such as dropdown menu.

Unlike most other jQuery plug-ins that wrap up all the styles in the javascript, this script is depended on a well maintains css file, not only for the style of the popup, but for it's placement as well.

The plugin will position of the element it is being called in against the anchor element passed in options. It will be place on a specific side of that via the placement. And here is the important bit, if placement: right, then the popover will be placed flush against anchor by it's most outer boundry, including the margin!

Getting Started

$('<div class="my-class">').popup();

Will simply Render:

<div class="popup-container">
	<div class="my-class"></div>
</div>

An example where you already have the markup in the DOM

$('.notification').popup();

Will turn this:

...
<aside>
	<div class="notification">This is a notification!</div>
</aside>

Into this:

...
<aside>
	<div class="popup-container">
		<div class="notification">This is a notification!</div>

API

Options

The object properties that can be passed on creation of a popup

Name Type Default Description
anchor jQuery | element | string null The element the popup will be positioned against. Can be a jQuery element, a dom element, or a query string. This element will also be given a reference to the element the popup was instantiated on. It is saved on anchor via data-popup-ref. ie: For
$element.popup({ anchor: $myButton });
then
$myButton.data('popup-ref') == $element
anchorPoint string '50%' Where on the anchor the popup will center too. If offset: 50% and showArrow: true, the arrow will point to this spot on the anchor. The percent range represents the edges of the anchor's border from end to end, with '50%' being the middle of the side you have choosen (based on placement). Accepts a percent/pixel combo: '50%-25px' will be 25px back from the middle of te anchor. For left and right, '0%' will be the very top of the anchor, and '100%' will be the very bottom of the anchor. For top and bottom, '0%' will be the very left of the anchor, and '100%' will be the very right of the anchor. Negative values and values over '100%' are ok. Invalid values will default to '50%'.
animate boolean false Setting animate to true will cause a series of classnames to be added and removed to popup-container to allow for animation effects on show/hide. The idea is to use a clever componation of position css properties / transforms and transitions. Where the popup is to be regularly positioned is where the transition will end on show. When you show the pop-up, these classes will be added/removed:
ani-start This will be where your popup begins on animation
ani-start ani-show ani-show should contain your transition
ani-show then remoing ani-start will transition your pop-up from it's start position to your show position. At the end of the transition, ani-show will be removed.

On hide, these classes will be added/removed:
ani-hide this will be your hide transiton
ani-hide ani-end ani-end is the position you want your hide transition to end on. This will trigger the transition on ani-hide. At the end of that transition, both these classes will be removed.
A lot of the time, ani-start and ani-end will be the same, as well as ani-show and ani-hide (see the fade example below). There are times when you want them to be different however, and that is why they are different classes (see the grow example below).
autoShow boolean true Display immediately after creation of the popup.
boundary number | string 10 An additional inner boundary inside the window. Used in tandam with collision. If set to 0, the window is the boundary. If set to 10, the boudary is 10 pixels inside the window on all 4 side. You can also pass in a string boundary: "50 10 10 10" that represents a different value for each side of the window (css style top-right-bottom-left).
classes string null Any class names you would like to add to the popup-container. ex: 'bubble', or for multiple: 'bubble fade'
collision string 'flip' There are 3 acceptable values. "flip", "fit", or "flipfit"
container jQuery | element | string null The element that the popup will be a child of. If left null, one of 2 things will happend:
  1. If the popup was already part of the DOM it will remain in it's current position
  2. If the popup is not already part of the DOM (just a documentFragment), it will be appended to the bottom of the body
destroyOnHide boolean false Destroy the popup on close. Usefull when you're creating the popup from a template and not an exisition dom element, and populating data points on creations. This allows you to get a fresh pop-up everytime.
offset string '50%' The popup is always positioned against the center of the anchor element. Offset displaces that position by either a percentage, pixels, or both. Percent is the percent of the popup's width or height (depends on if the placement is left/right or top/bottom). If set to '50%' the middle of the popup will align with the middle of anchor. If set to '30px' the popup will align with the middle of anchor from 30px from the Top/Left of the popup. If you want 30px from the Bottom/Right of the popup, you would set this too '100%-30px'. See the offset example below to play around with some values.
placement string 'right' This determines how the popup is placed against anchor. Your valid options are 'top', 'right', 'bottom', or 'left'. If you are no placing the popup against anchor, you can also pass in 'middle' or 'free'. 'Middle' will position the popup in the dead center of the current view port. It will not be fixed, so the popup will scroll. 'Free' skips the positioning process altogether. 'Free' is the default if you pass an invalid position, and is only recomended if you want to wire up your own positioning (say for when you want to just create floating elements that are draggable). The placement will also be added as a classname to popup-container.
showArrow boolean true When set to true, the markup for the arrow will be added to popup-container. The arror element will always be position to the middle of anchor, regardless of what offset is set too. See the offset example below for a visual.
showClose boolean true When set to true, the markup for the close button will be added to popup-container. The position of this element is completely left up to your css. It is automatically binded to trigger the method 'close' on click.
within jQuery | element | string $(window) The within option selects what the popup is constrained by in terms of its collision. By default it is the browser window, however it can be any DOM element. In general it will be the site's or module's container element that the popup will exist in. In the example below, you can choose within to be either the browser window, or this site's <main.container> tag.

Methods

There are methods that can be called for your programming convenience.

Name Use Description
show $element.popup('show'); Reveal the popup. The classname showing is added to popup-container once shown (if animate = true it will be added at the and of the show transition.
hide $element.popup('hide'); Hide the popup/ The classname showing will be removed immediately from popup-container when this method is called.
toggle $element.popup('toggle'); Toggle the popup
destroy $element.popup('destroy'); Completely destroy the popup instance and remove the popup-container markup (and it's children) from the DOM. All events related to the popup are also removed. WARNING! If you instantiated the popup on markup that already existed on the DOM, it will be remove with this method, and not be placed back in it's original place.
position $element.popup('position'); Rerun the popup's position function. Useful for when you add/remove content from the popup and you keep the popup align properly on it's anchor.
setContent $element.popup('setContent', content); Sets the content of $element. This method will remove all children of $element and replace them with content. This method is very handy when you want to populate the popup with fresh content loaded via ajax. content can be a jquery object, an HTML Element, or a html string. It does use $.fn.append(), so if you pass it a Node that already exists somewhere else in the DOM, it will be moved and become a child of the popup (this is ideal for when you are using a DocumentFragment you created from an html string brought in via ajax). The popup will automatically re-position itself on this call (to account for width/height changes)

Properties

Get only properties for your programming convenience

Name Return Type Description
$popupContainer jQuery Returns a jQuery of the pop-container your popup has been nested in.
isShowing boolean Returns show/hide state of the popup.

Events

This are the events you can hook into. All the Events are called on the element .popup() was called upon.

$('#message').popup({...});
$('#message').on('show.popup', function() { ... });
Name Description
create.popup Triggered at the end of the instantiation, and before showing if autoOpen = true.
show.popup Triggered right as the popup begins the process of showing.
shown.popup Triggered once the popup has fully closed. Generally this and show.popup will happen within an instance of each other, except when animate = true.
positioned.popup Triggered after the popup has been positioned.
hide.popup Triggerd right as the popup begins the process of hiding.
hidden.popup Triggered once the popup has fully closed. Like show and shown, hide and hidden will happen within an instance of each other, except when animate = true.
destroy.popup Triggered when destroying the popup. This trigger happens directly before $popupContainer is removed.

Examples

Note: all of the following examples are always passing the option classes: 'bubble' to add to the visual effect of the demo. I am leaving that out of the code examples because I only want to show what options are needed for the actually example. I am also leaving out anchor, which for all examples are the buttons that show/hide the popup.

Common Use Case

The most common use case is a click event on a button opens a popup. Here is an example of when the popup is instantiated on page load:

var $popup = $('#common-popup').popup({
	anchor: '#common-button',
	autoShow: false,
	classes: 'bubble',
	showArrow: true,
	showClose: true
});

$('#common-button').on('click', function() {
	$popup.popup('toggle');
});
I am a common example of a popup

And here is an example when you want to lazy instantiate the popup, but still have the same functinality:

var html = '<div id="lazy-popup">I am a lazy loaded popup.</div>';
$('#lazy-button').on('click', function() {
	var $this = $(this);

	// since #common-popup will have data('popup-ref')
	// because it's the anchor,
	// we can see if the popup has been instanciated yet or not
	if (!$this.data('popup-ref'))
		$(html).popup({
			anchor: $this,
			classes: 'bubble',
			showArrow: true,
			showClose: true
		});
	}
	// and once the pop-up has been instantiated,
	// we then just have the button toggle the popup
	else {
		$this.data('popup-ref').popup('toggle');
	}
});

jqXHR

$('#jqXHR-test').on('click', function() {

	var echo = $('#jqXHR-echo').val();

	var $ajax = $.ajax({
		type: 'GET',
		url: 'http://jsfiddle.net/echo/jsonp/',
		data: {
			html: '<div class="popup"> ... </div>',
			echo: echo,
			delay: 3
		},
		dataType: 'jsonp'
	}).done(function(data) {
		$echoPopup.popup('setContent', data.html);
	});

	var $echoPopup = $.popup($ajax, {
		anchor: '#jqXHR-test',
		classes: 'bubble',
		collision: false,
		placement: 'right',
		destroyOnHide: true
	});
});
